YANDEX Auth:


ANDROID:

1. При регистрации приложения в Яндекс.OAuth добавляем свой Callback URI (для этого ставим галку Веб-сервисы).
Этот URI состоит из двух частей = "Part1://Part2" 
При регистрации приложения Part1 мы можем придумать сами. А вот Part2 лучше самостоятельно не придумывать и воспользоваться
кнопкой "Подставить URL для разработки".

2. Part1 определяет какая Activity вашего приложения будет запущена для обработки этого вызова. Part1 должна быть 
указана в качестве значения DataSchemes=new[]{"Part1"} при описании Activity в перехватчике вызовов.
Можно под каждого дата провайдера (GD,YD,MRD и т.д.) создать отдельную отдельную схему Part1 и вызывать 
свою отдельную Activity. 
А можно сделать одну схему Part1 и вызывать одну Activity, и различать дата-провайдера по 
Part2 через разбор входящей строки в Activity.OnCreate().  Странно, но прописывать для Activity  
DataPaths=new[]{"Part1 for GD", "Part1 for YD",...} не обязательно.

3. Учитывая вышесказанное, дефолтный Redirect URI (Call-back URI) нам не нужен.
